# 06. Astro 프로젝트 만들기

## 목표

작업 폴더 안에 [Astro](https://astro.build/) 개인 홈페이지 프로젝트를 만듭니다.

## 왜 하는가

Astro는 개인 홈페이지, 블로그, 프로젝트 소개처럼 콘텐츠 중심 사이트에 잘 맞습니다. 오늘은 복잡한 앱이 아니라 내 소개와 프로젝트를 보여줄 첫 버전을 만들기 때문에 Astro를 사용합니다.

## 쉬운 설명

| 단어 | 의미 |
|---|---|
| Astro | 개인 홈페이지나 블로그를 만들기 좋은 웹 프레임워크 |
| package.json | 프로젝트 명령어와 필요한 패키지 정보가 들어 있는 파일 |
| src | 실제 페이지와 컴포넌트가 들어가는 폴더 |
| public | 이미지나 정적 파일을 넣는 폴더 |

## SwiftUI와 비교해서 이해하기

SwiftUI를 조금 봤다면 아래처럼 비교해볼 수 있습니다.

| Astro 프로젝트 | SwiftUI에서 떠올릴 쉬운 개념 |
|---|---|
| `src/pages/index.astro` | 첫 화면 역할을 하는 `ContentView` |
| `.astro` 파일 | 화면을 구성하는 `View` 파일 |
| `public` 폴더 | 앱에 넣는 이미지나 리소스 |
| `package.json` | 프로젝트 명령어와 패키지 정보를 적어둔 파일 |

완전히 같은 구조는 아닙니다.
오늘은 단순히 **Astro에도 화면 파일과 리소스 폴더가 있다**는 정도로 이해하면 충분합니다.

## 공통 프롬프트

```text
이 폴더에 Astro 기반 개인 홈페이지 프로젝트를 만들고 싶어.

목표:
- 개인 홈페이지 첫 버전 만들기
- GitHub Pages에 배포할 예정
- 오늘은 Home 페이지만 있어도 됨
- 나중에 About / Projects / Posts로 확장할 수 있게 시작

조건:
- 먼저 어떤 명령어와 파일 구조로 만들지 계획을 설명해줘.
- 내가 승인하기 전에는 실행하지 마.
- 새 라이브러리는 필요한 경우에만 제안하고, 설치 전 반드시 물어봐.
- 프로젝트 생성 후 어떤 파일이 생겼는지 설명해줘.
- 가능하면 SwiftUI의 ContentView나 View 파일과 비교해서 쉽게 설명해줘.
- 단, SwiftUI의 상태 관리나 앱 아키텍처 같은 어려운 개념은 설명하지 마.
```
