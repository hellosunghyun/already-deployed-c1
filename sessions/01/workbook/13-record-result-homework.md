# 13. 결과 기록과 다음 숙제

## 목표

오늘 만든 결과와 막힌 지점을 기록합니다.

## 왜 하는가

배포는 끝이 아니라 이후 수정의 시작입니다. 오늘 만든 링크와 막힌 지점을 기록해야 다음 세션에서 이어갈 수 있습니다.

## 결과 기록 폼

워크숍 플랫폼에서 미리 알 수 있는 값은 자동으로 채워지는 것이 좋습니다.

자동 입력되면 좋은 정보:

- 닉네임
- 사용한 AI 도구
- 운영체제
- 완료한 체크박스
- 마지막으로 완료한 단계
- 틀린 퀴즈

참가자가 직접 입력할 정보:

```text
GitHub 저장소 URL:
GitHub Pages URL:
오늘 막힌 지점:
다음 세션에서 이어가고 싶은 것:
```

## 체크박스

- [ ] GitHub 저장소 URL을 기록했다.
- [ ] GitHub Pages URL을 기록했다. 또는 실패 로그를 기록했다.
- [ ] 오늘 막힌 지점을 기록했다.
- [ ] 다음 세션에서 고치고 싶은 것을 하나 적었다.

## 다음 세션 전 숙제

다음 세션에서는 Home / About / Projects / Posts 구조를 만듭니다.

아래를 준비해오면 훨씬 수월합니다.

- 나를 소개하는 한 문장
- 홈페이지에 넣고 싶은 섹션 3개
- 보여주고 싶은 프로젝트 / 경험 3개
- 마음에 드는 개인 홈페이지 링크 1개
- 오늘 이해 안 된 단어 3개

## 다음 세션과의 연결

오늘 만든 것은 **인터넷에 올라간 첫 버전**입니다.

다음 세션부터는 이 사이트를 이렇게 키웁니다.

```text
1회차: 일단 github.io 링크 만들기
2회차: Home / About / Projects / Posts 구조 만들기
3회차: 화면을 보며 디자인 수정 요청하기
4회차: Notion을 콘텐츠 관리 도구처럼 연결하기
5회차: README와 운영 방법 정리하기
```

## 마무리 문장

오늘 `github.io` 링크가 열렸다면 아주 좋습니다.
열리지 않았더라도 GitHub 저장소 URL이나 실패 로그가 있다면 다음 세션에서 이어갈 수 있습니다.

오늘 배운 핵심은 이것입니다.
