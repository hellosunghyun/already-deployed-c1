# 08. Home 페이지 초안 만들기

## 목표

기본 템플릿을 내 개인 홈페이지 첫 화면처럼 바꿉니다.

## 왜 하는가

GitHub Pages에 배포했을 때, 최소한 **내 사이트라는 느낌**이 나야 합니다. 완성도는 낮아도 괜찮습니다. 오늘은 초안이면 충분합니다.

## 쉬운 설명

처음부터 모든 내용을 완벽히 쓸 필요는 없습니다.

오늘 Home 페이지에는 아래 정도만 있으면 됩니다.

- 이름 또는 닉네임
- 한 줄 소개
- 나중에 About / Projects / Posts로 확장할 자리
- 너무 복잡하지 않은 기본 디자인

## 공통 프롬프트

```text
현재 Astro 프로젝트의 첫 화면을 개인 홈페이지 초안처럼 바꿔줘.

목표:
- 첫 화면에서 내 개인 홈페이지라는 느낌이 나야 함
- 이름 또는 닉네임을 넣을 수 있어야 함
- 한 줄 소개가 있어야 함
- 나중에 About / Projects / Posts로 확장할 수 있는 섹션이 있어야 함

디자인:
- 흰 배경
- 넓은 여백
- 큰 제목
- 읽기 쉬운 고딕 느낌
- 과한 애니메이션 없음

제약:
- 새 라이브러리 추가하지 마
- Astro 구조를 유지해
- 관련 없는 파일은 수정하지 마

수정 후:
1. 변경한 파일 목록
2. 왜 바꿨는지
3. 브라우저에서 확인할 주소
를 알려줘.
```

## Codex에서 브라우저 확인 요청

```text
@browser 현재 localhost 화면을 확인해줘.
Home 페이지가 개인 홈페이지 초안처럼 보이는지 확인하고, 이상한 점이 있으면 알려줘.
```

브라우저 도구가 없으면 직접 Safari / Chrome에서 확인하세요.

## 체크박스

- [ ] 홈 화면에 내 이름 또는 임시 이름이 보인다.
- [ ] 한 줄 소개가 보인다.
- [ ] 기본 템플릿 그대로가 아니다.
- [ ] 브라우저에서 수정 결과를 확인했다.
- [ ] AI가 변경 파일과 이유를 설명했다.

## Ready Gate

다음 단계로 넘어가기 전에 아래가 되어 있어야 합니다.

- 홈 화면이 기본 템플릿 그대로가 아닙니다.
- 내 이름 또는 임시 이름이 보입니다.
- 브라우저에서 수정 결과를 확인했습니다.

## 힌트

### Hint 1

문장이 준비되지 않았다면 임시 문장으로 시작하세요. 예: `AI와 함께 만드는 첫 개인 홈페이지입니다.`

### Hint 2

오늘은 글이 완벽하지 않아도 됩니다. 2회차에서 About / Projects / Posts 구조를 만들며 다시 바꿀 수 있습니다.

### Hint 3

AI가 디자인을 너무 많이 바꾸면 이렇게 말하세요.

```text
새 라이브러리 추가하지 말고, 흰 배경과 큰 제목 중심으로 더 단순하게 다시 정리해줘.
```
