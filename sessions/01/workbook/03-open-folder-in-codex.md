# 03. Codex에서 프로젝트 폴더 열기

## 목표

실습할 프로젝트 폴더를 만들고, 선택한 코딩 에이전트가 그 폴더를 작업 범위로 보게 합니다.

## 왜 하는가

AI가 아무 곳이나 파일을 만들면 나중에 찾기 어렵습니다. 프로젝트 폴더는 **오늘 작업할 안전한 범위**입니다.

## Mac에서 폴더 만들기

1. Finder를 엽니다.
2. 원하는 위치에 새 폴더를 만듭니다. 예: `배포했는데요-1회차`
3. 폴더 이름은 가능하면 영어 소문자와 하이픈을 추천합니다. 예: `deployed-session-01`

## Codex에서 폴더 열기

1. [Codex](https://chatgpt.com/codex)를 엽니다.
2. 새 작업 또는 새 프로젝트를 시작합니다.
3. 방금 만든 폴더를 엽니다.
4. Codex가 해당 폴더 안의 파일을 읽고 수정할 수 있는 상태인지 확인합니다.

정확한 버튼 이름은 앱 버전에 따라 조금 다를 수 있습니다. 버튼 이름이 다르면 AI에게 이렇게 물어보세요.

```text
나는 Codex에서 프로젝트 폴더를 열고 싶어.
지금 화면에서 어떤 버튼을 눌러야 하는지 단계별로 안내해줘.
```

## Claude Code 사용자는

[Claude Code](https://docs.claude.com/en/docs/claude-code)에서 작업 폴더를 열거나, 프로젝트 디렉터리를 기준으로 세션을 시작합니다. 나중에 규칙 파일은 `CLAUDE.md`를 사용합니다.

## Antigravity 사용자는

[Antigravity](https://codelabs.developers.google.com/getting-started-google-antigravity)에서 workspace 또는 project folder를 열고, Agent가 해당 폴더를 기준으로 작업하게 합니다.

## 체크박스

- [ ] 실습용 폴더를 만들었다.
- [ ] 선택한 AI 도구에서 해당 폴더를 열었다.
- [ ] AI가 이 폴더를 기준으로 작업한다는 것을 확인했다.

## Ready Gate

다음 단계로 넘어가기 전에 아래가 되어 있어야 합니다.

- 작업 폴더가 있습니다.
- AI 도구에서 그 폴더가 열려 있습니다.
- 앞으로 생성되는 파일이 그 폴더 안에 생길 예정입니다.

되어 있나요?
