# 00. 워크북 사용법

## 이 문서는 어떻게 읽나요?

이 워크북은 일반 강의 노트가 아닙니다. 각 파일은 **하나의 실습 단계**입니다.

각 단계는 아래 구조를 가집니다.

- **목표**: 이 단계가 끝나면 무엇이 생기는가
- **왜 하는가**: 이걸 하지 않으면 다음 단계가 왜 어려운가
- **쉬운 설명**: 처음 보는 단어를 짧게 풀어 설명
- **AI에게 보낼 프롬프트**: 그대로 복사해서 사용 가능
- **체크박스**: 내가 실제로 했는지 확인
- **Ready Gate**: 다음 단계로 넘어가도 되는지 확인
- **힌트**: 바로 정답을 주지 않고 단계적으로 도움
- **퀴즈**: 헷갈릴 수 있는 개념 확인

## 중요한 규칙

### 1. 모르면 멈추지 말고 AI에게 질문합니다

오늘 범위 안에서 나올 수 있는 질문은 AI가 꽤 높은 신뢰도로 도와줄 수 있습니다.
단, 질문할 때는 맥락을 줘야 합니다.

나쁜 질문:

```text
안 돼요.
```

좋은 질문:

```text
나는 1회차 워크북의 "환경 확인과 설치" 단계에 있어.
Codex를 쓰고 있고, Mac에서 진행 중이야.
node -v를 실행했는데 command not found가 나왔어.
무엇을 설치해야 하는지, 내가 직접 해야 하는 것과 네가 실행해도 되는 것을 나눠서 알려줘.
```

### 2. AI가 실행하기 전에 계획을 먼저 받습니다

항상 이렇게 요청합니다.

```text
아직 실행하지 말고, 먼저 계획을 설명해줘.
내가 승인하면 실행해줘.
```

### 3. 터미널을 무서워하지 않아도 됩니다

[터미널](https://support.apple.com/guide/terminal/welcome/mac)은 Mac에서 명령어를 실행하는 앱입니다. 오늘은 명령어를 외우지 않아도 됩니다. Codex가 실행 계획을 설명하고 필요한 명령어를 도와줄 것입니다.

### 4. 오늘 배포 목표는 GitHub Pages입니다

오늘 최종 링크는 아래 형태입니다.

```text
https://<github-username>.github.io
```
