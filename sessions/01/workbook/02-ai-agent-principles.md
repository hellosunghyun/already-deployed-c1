# 02. AI 에이전트 사용 원칙

## 목표

코딩 에이전트를 단순 채팅봇처럼 쓰지 않고, **작업 파트너**처럼 쓰는 기본 원칙을 익힙니다.

## 왜 하는가

AI에게 그냥 "홈페이지 만들어줘"라고 말하면 빠르게 무언가를 만들 수는 있습니다. 하지만 무엇을 왜 만들었는지 알기 어렵고, 잘못된 설정이 들어가도 놓치기 쉽습니다.

오늘은 아래 루프를 계속 사용합니다.

```text
목표 제시
→ 계획 요청
→ 승인
→ 실행
→ 검증
→ 피드백
```

## 좋은 요청의 구조

```text
목표:
무엇을 만들고 싶은지

현재 상태:
지금 어디까지 되었는지

제약:
하지 말아야 할 것

완료 기준:
무엇이 되면 성공인지

검증:
어떻게 확인할 것인지
```

## 오늘 계속 쓸 기본 문장

```text
아직 실행하지 말고, 먼저 계획을 설명해줘.
내가 승인하면 실행해줘.
```

## AI에게 자주 말해야 할 제약

```text
- 새 라이브러리는 내 승인 없이 추가하지 마.
- 관련 없는 파일은 수정하지 마.
- secret, token, password, API key를 코드에 쓰지 마.
- 삭제 명령은 실행 전에 반드시 물어봐.
- 변경 후에는 어떤 파일을 왜 바꿨는지 알려줘.
- 가능하면 브라우저나 로그로 결과를 확인해줘.
```

## 퀴즈

### 질문

AI에게 코딩을 맡길 때 가장 먼저 요청해야 하는 것은?

A. 아무 설명 없이 바로 실행하기  
B. 먼저 작업 계획을 설명하게 하기  
C. 최대한 많은 라이브러리를 설치하게 하기  
D. 모든 파일을 한 번에 바꾸게 하기

<details>
<summary>정답 보기</summary>

**정답**

B

**해설**

먼저 계획을 받아야 AI가 무엇을 하려는지 알 수 있습니다. 계획을 확인한 뒤 실행하면 실수를 줄이고, 잘못된 방향으로 가는 것을 빨리 막을 수 있습니다.

</details>
