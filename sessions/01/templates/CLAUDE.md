# CLAUDE.md

이 파일은 Claude Code가 이 프로젝트에서 참고할 규칙입니다.

## 프로젝트 목표

- 이 프로젝트는 Astro 기반 개인 홈페이지입니다.
- 최종 배포 목표는 GitHub Pages의 `github.io` 주소입니다.
- 오늘은 Home 페이지 첫 버전을 만들고 배포하는 것이 목표입니다.
- 이후 About / Projects / Posts 구조로 확장할 예정입니다.

## 작업 원칙

- 새 라이브러리는 사용자 승인 없이 추가하지 않습니다.
- 관련 없는 파일은 수정하지 않습니다.
- secret, token, password, API key를 코드에 쓰지 않습니다.
- 삭제 명령은 실행 전에 반드시 사용자에게 물어봅니다.
- 수정 후 변경한 파일 목록과 변경 이유를 설명합니다.
- 가능하면 `npm run build`로 확인합니다.
- preview 또는 browser로 확인할 수 있는 작업은 화면에서 결과를 확인합니다.

## Git / commit 규칙

변경사항이 생기면 자주 commit합니다.
그 이유는 AI가 빠르게 많이 수정할 수 있으므로, 실수했을 때 이전 상태로 돌아가기 쉽게 하기 위해서입니다.
단, commit과 push는 변경사항 요약을 먼저 보여주고 사용자 승인을 받은 뒤 진행합니다.

commit 메시지는 아래 형식을 사용합니다.

- `feat: 한국어설명`
- `fix: 한국어설명`
- `chore: 한국어설명`
- `docs: 한국어설명`

예시:

- `chore: 초기 Astro 프로젝트 생성`
- `feat: 홈 화면 초안 추가`
- `fix: GitHub Pages 배포 설정 수정`
- `docs: Claude 작업 규칙 추가`

## 사용자가 초보자일 수 있음

사용자는 터미널, npm, Git, GitHub Pages를 처음 볼 수 있습니다.
따라서 명령어를 실행하기 전에 무엇을 하는 명령인지 짧게 설명합니다.
