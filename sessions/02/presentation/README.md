# 2회차 발표자료 인덱스

이 발표자료는 Markdown 뷰어에서 슬라이드처럼 한 장씩 넘겨가며 설명하기 위한 자료입니다.
각 슬라이드는 별도 파일로 나누었습니다.

## 발표 시간 목표

권장 발표 시간은 25~30분입니다.
1회차보다 개념이 조금 더 개발 쪽으로 들어가지만, 목표는 문법 강의가 아닙니다.
참가자가 워크북에서 만날 단어를 미리 익히고, 왜 이 구조를 만드는지 이해하게 하는 것이 핵심입니다.

## 발표자료의 역할

이 발표자료는 실습 진도를 줄이기 위한 자료가 아닙니다.
참가자들이 실습 중에 같은 단어를 쓰도록 만드는 공통 개념 세팅 자료입니다.

특히 아래 개념은 혼자만 아는 것보다 모두가 같이 알고 있어야 실습 중 질문과 피드백이 쉬워집니다.

- 페이지와 섹션
- 파일 기반 라우팅
- Layout
- Navigation
- Component
- 데이터와 화면 분리
- Posts를 나중에 외부 콘텐츠 소스로 확장하는 이유

따라서 발표자료는 25~30분 기준으로 유지합니다.
다만 발표 중 코드를 자세히 설명하기보다, 워크북에서 다시 만날 단어를 먼저 익히는 데 집중합니다.

발표자료는 오늘 실습에서 만날 개념을 미리 공유하기 위한 자료입니다.
모든 참가자가 발표 내용을 즉시 코드로 구현해야 한다는 뜻은 아닙니다.
실제 구현 속도는 워크북의 최소 성공 기준과 Ready Gate를 기준으로 판단합니다.

## 30분 진행 리듬

| 시간 | 슬라이드 | 진행 방식 |
|---|---|---|
| 0:00-0:05 | 01-02 | 오늘의 성공 기준과 최소 목표 확인 |
| 0:05-0:12 | 03-05 | 왜 한 페이지를 여러 페이지로 나누는지 설명 |
| 0:12-0:18 | 06-06A | 페이지와 섹션 구분을 짧게 연습 |
| 0:18-0:24 | 07-09A | URL, Layout, Component를 쉬운 비유로 설명 |
| 0:24-0:28 | 10-11A | Notion을 미루는 이유와 Codex 데모 흐름 |
| 0:28-0:30 | 12 | 워크북으로 전환 |

각 슬라이드를 길게 읽기보다, 참가자에게 "이건 페이지인가요, 섹션인가요?"처럼 짧은 질문을 던지며 진행합니다.
