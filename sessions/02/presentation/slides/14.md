# Slide 14. 페이지와 섹션을 구분해봅니다

아래 요청을 보고 페이지인지 섹션인지 판단합니다.

| 요청 | 답 |
|---|---|
| About 화면을 따로 만들고 싶다 | 페이지 |
| Home 안에 관심 분야 소개를 넣고 싶다 | 섹션 |
| Projects 목록을 주소로 열고 싶다 | 페이지 |
| 첫 화면 아래에 연락처 블록을 넣고 싶다 | 섹션 |

```text
페이지를 만들어줘 = 주소가 따로 생김
섹션을 만들어줘 = 한 페이지 안에 구역이 생김
```

<details><summary>슬라이드 모습과 강조점</summary>

**슬라이드 모습**

- 4개 예시 표를 화면 중심에 둡니다.
- 아래에는 요청 문장의 차이를 한 줄씩 보여줍니다.
- 참가자가 먼저 답해볼 수 있게 여백을 둡니다.

**강조해야 하는 내용**

- 정답을 바로 말하기보다 먼저 질문합니다.
- 용어 암기보다 예시 판단이 중요합니다.
- 이 구분은 뒤의 라우팅 설명과 연결됩니다.

</details>

<details><summary>발표자 노트</summary>

여기서는 진행자가 바로 답을 말하지 말고 참가자에게 먼저 물어봅니다.
2~3분 정도만 써도 충분합니다.

About 페이지와 Home 안의 About 섹션이 다르다는 감각만 잡으면 됩니다.
이제 주소가 따로 생기는 페이지가 Astro에서는 파일과 어떻게 연결되는지로 넘어갑니다.

</details>
