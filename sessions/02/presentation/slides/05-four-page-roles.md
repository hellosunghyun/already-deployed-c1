# Slide 05. Home / About / Projects / Posts

각 페이지는 역할이 다릅니다.

단, 이 구분은 정답이라기보다 **이번 수업에서 쓰는 기준**입니다.
개인 홈페이지의 성격에 따라 나중에 바꿔도 됩니다.

| 페이지 | 역할 |
|---|---|
| Home | 첫인상과 안내 |
| About | 개인이나 회사 소개처럼 다양한 내용을 이어서 설명 |
| Projects | 비슷한 형식의 카드와 상세정보를 반복해서 보여줌 |
| Posts | 글 중심 콘텐츠를 모아 보여줌 |

나누는 이유는 내용의 성격이 다르기 때문입니다.

```text
About = 소개 흐름
Projects = 같은 형식의 사례 목록
Posts = 글 중심 기록
```

---

## 오늘 다루는 기본 형태

- Home: 이름, 한 줄 소개, 주요 링크
- About: 짧은 소개와 관심사
- Projects: 클릭 가능한 프로젝트 카드 3개
- Posts: 샘플 글 목록 1~2개

**슬라이드 화면 구성**

- 표: Home / About / Projects / Posts의 역할
- 짧은 비유: `About = 소개 흐름`, `Projects = 같은 형식의 사례 목록`, `Posts = 글 중심 기록`
- 기본 형태: Home, About, Projects, Posts에 오늘 다룰 최소한의 예시

**말할 내용**

오늘은 완성된 포트폴리오 문장을 쓰는 날이 아닙니다.
임시 문장으로 시작해도 됩니다.
그리고 이 페이지 구분을 정답처럼 말하지 않습니다.

여기서는 이렇게 말해도 좋습니다.

```text
이건 일단 오늘 수업에서 쓰는 기준입니다.
나중에 내 홈페이지 성격에 맞게 바꿔도 됩니다.
그래도 About / Projects / Posts를 나누는 데는 이유가 있습니다.
About은 소개 글처럼 다양하게 이어지고, Projects는 비슷한 카드와 상세정보가 반복되고, Posts는 글 중심 콘텐츠로 쌓입니다.
오늘 문장이 완벽하지 않아도 됩니다.
나중에 바꿀 수 있는 자리를 먼저 만들어두는 게 더 중요합니다.
```

여기서 "기본 형태"는 오늘 다루는 범위입니다.
앞 슬라이드의 최소 성공 기준과 헷갈리지 않게, 시간이 부족하면 Projects / Posts는 데이터 초안이나 막힌 지점 기록만 남겨도 된다고 한 번 더 말합니다.
