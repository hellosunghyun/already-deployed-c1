# Slide 22. Posts를 미리 만드는 이유

나중에는 글 데이터를 다른 콘텐츠 소스에서 가져올 수 있습니다.

하지만 오늘은 API를 붙이지 않습니다.

오늘은 먼저 화면이 기대하는 글 모양을 만듭니다.

```text
샘플 글 데이터
→ Posts 페이지
→ 나중에 외부 데이터 응답을 같은 모양으로 변환
```

오늘 하지 않는 것:

- database 만들기
- Integration token 만들기
- secret 설정하기
- API 호출하기

<details><summary>슬라이드 모습과 강조점</summary>

**슬라이드 모습**

- `샘플 글 데이터 → Posts 페이지 → 나중에 변환` 흐름을 중앙에 둡니다.
- 오늘 하지 않는 것은 아래에 짧게 둡니다.
- `API를 붙이지 않습니다` 메시지가 분명해야 합니다.

**강조해야 하는 내용**

- Posts의 목표는 블로그 시스템 완성이 아닙니다.
- 오늘은 글 목록이 화면에서 어떤 데이터 모양으로 쓰일지 정합니다.
- 외부 서비스, token, secret은 오늘 건드리지 않습니다.

</details>

<details><summary>발표자 노트</summary>

오늘 Posts를 만든다고 해서 외부 콘텐츠 소스를 연결하는 것은 아닙니다.
먼저 글 목록이 화면에서 어떤 모양으로 쓰일지 잡아둡니다.

나중에 외부 데이터가 들어와도 결국 화면에서는 `title`, `slug`, `date`, `summary`, `tags`처럼 단순한 정보로 쓰이게 됩니다.
그래서 오늘은 샘플 JSON 데이터로 화면이 기대하는 모양을 먼저 만듭니다.

token, secret, database 설정으로 빠지면 수업의 초점이 흐려집니다.
오늘은 구조를 잡는 회차라고 분명히 말합니다.

</details>
