# Slide 25. Codex와 작업할 때의 흐름

오늘 데모는 전체 구현이 아닙니다.

안전하게 요청하고 확인하는 흐름을 봅니다.

```text
현재 상태 확인
→ 계획 받기
→ 수정할 파일 확인
→ 승인 후 진행
→ Browser 확인
→ build 확인
→ 결과 기록
```

먼저 할 말:

```text
아직 파일을 수정하지 말고, 먼저 계획을 설명해줘.
내가 승인하면 진행해줘.
```

<details><summary>슬라이드 모습과 강조점</summary>

**슬라이드 모습**

- 작업 흐름을 한 줄짜리 단계로 보여줍니다.
- 아래에는 반드시 먼저 말할 프롬프트를 코드블록으로 크게 둡니다.
- 데모를 구현 시연이 아니라 협업 흐름 시연으로 보이게 합니다.

**강조해야 하는 내용**

- 빠르게 코드를 쓰는 장면보다 계획 확인이 중요합니다.
- 수정 파일이 많아질수록 먼저 계획을 받아야 합니다.
- Browser 확인과 build 검증까지가 하나의 요청 흐름입니다.

</details>

<details><summary>발표자 노트</summary>

데모에서 가장 강조할 것은 코드를 빠르게 쓰는 장면이 아닙니다.
AI가 바로 파일을 고치기 전에 계획을 말하게 하는 습관입니다.

2회차는 새 페이지, Layout, Navigation, 데이터 파일처럼 건드릴 파일이 늘어납니다.
그래서 먼저 계획을 설명해달라고 요청하지 않으면 참가자가 어떤 일이 일어나는지 따라가기 어렵습니다.

참가자가 봐야 할 흐름은 현재 상태 확인, 계획 요청, 승인, Browser 확인, build 확인, 기록입니다.

</details>
