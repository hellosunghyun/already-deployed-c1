# Slide 12. 이제 워크북으로 갑니다

오늘 워크북은 아래 순서로 진행합니다.

1. 1회차 결과 확인
2. 현재 프로젝트 상태 점검
3. 페이지 구조 만들기
4. Layout과 Navigation 만들기
5. Home / About / Projects / Posts 채우기
6. 브라우저와 build로 확인
7. commit / push / deploy
8. AGENTS.md를 현재 구조에 맞게 갱신
9. 다음 세션 전 홈페이지 레퍼런스 찾기

---

## 실습 중 규칙

모르면 멈추지 말고, 아래 정보를 담아 AI에게 질문합니다.

- 지금 워크북 단계
- 현재 프로젝트 상태
- 방금 실행한 명령어
- 에러 메시지
- 원하는 결과

**슬라이드 화면 구성**

- 워크북 진행 순서 1~9
- 실습 중 질문할 때 담을 정보 5개
- 화면 마지막에는 “각자 속도대로 Ready Gate를 기준으로 진행”이 느껴지면 좋습니다.

**말할 내용**

여기서 발표를 종료합니다.
이후에는 진행자가 계속 강의하기보다, 참가자가 워크북을 보며 직접 움직이게 합니다.

발표자료는 실습 진도표가 아닙니다.
발표자료는 오늘 만날 개념을 같은 언어로 이해하기 위한 준비입니다.
실제 실습은 각자의 속도와 워크북의 Ready Gate를 기준으로 진행합니다.

중간중간 전체에게 짧게만 확인합니다.

```text
지금 몇 단계에 있나요?
로컬 화면은 열리나요?
build 결과나 실패 로그가 있나요?
```

commit까지 못 가도 괜찮습니다.
AGENTS.md를 파일로 수정하지 못해도 괜찮습니다.
마지막에는 각자 현재 상태, AGENTS.md 갱신 여부 또는 갱신 프롬프트, 다음 질문, 다음 세션에 가져올 홈페이지 레퍼런스를 남기게 합니다.
발표를 마칠 때는 "이제부터는 속도가 다를 수 있다"고 명시합니다.
빠른 사람은 Projects / Posts와 build까지 가고, 느린 사람은 열린 URL과 실패 로그를 남기는 것이 목표입니다.
