# Slide 23. 만든 뒤에는 검증합니다

AI가 파일을 만들었다고 끝이 아닙니다.

오늘 검증 흐름:

```text
npm run dev
→ Browser에서 /, /about, /projects, /posts 확인
→ Navigation 클릭
→ Projects 상세 링크 확인
→ npm run build
→ build 결과 또는 실패 로그 기록
```

```text
dev = 내 컴퓨터에서 화면 확인
build = 배포 전 검증
```

<details><summary>슬라이드 모습과 강조점</summary>

**슬라이드 모습**

- 검증 흐름을 단계형 코드블록으로 보여줍니다.
- `dev`와 `build` 차이를 두 줄로 정리합니다.
- 실패 로그도 결과라는 메시지를 노트에서 연결합니다.

**강조해야 하는 내용**

- 파일 변경만으로 완료가 아닙니다.
- Browser 확인과 build는 둘 다 필요합니다.
- 실패하면 실패 로그를 남기는 것이 오늘의 최소 성공 기준입니다.
- build가 끝난 뒤 commit / push / deploy 확인으로 이어집니다.

</details>

<details><summary>발표자 노트</summary>

AI가 파일을 만들었다고 바로 끝내지 않습니다.
브라우저에서 실제로 페이지가 열리는지 확인하고, build가 되는지 확인해야 합니다.

`npm run dev`는 내 컴퓨터에서 작업 중인 화면을 보는 명령입니다.
`npm run build`는 배포 가능한 결과물을 만들 수 있는지 확인하는 명령입니다.

화면은 보이는데 build가 실패할 수 있습니다.
그래서 build 결과나 실패 로그를 반드시 남기게 합니다.

build가 성공하면 다음 단계에서 변경사항을 저장하고 GitHub에 올리는 흐름으로 넘어갑니다.
build가 실패하면 commit보다 실패 로그 기록이 먼저입니다.

</details>
