# Slide 15. Astro에서는 파일이 주소가 됩니다

[Astro](https://astro.build/)에서는 `src/pages` 안의 파일이 URL이 됩니다.

```text
src/pages/index.astro     → /
src/pages/about.astro     → /about
src/pages/projects.astro  → /projects
src/pages/posts.astro     → /posts
```

오늘 필요한 감각은 단순합니다.

파일 이름을 잘 정하면 방문자가 열 수 있는 주소가 생깁니다.

<details><summary>슬라이드 모습과 강조점</summary>

**슬라이드 모습**

- 파일 경로와 URL 매핑을 가장 크게 보여줍니다.
- 하단에는 `파일 이름을 잘 정하면 주소가 생김` 문장을 둡니다.
- 서버나 고급 라우팅 설명은 넣지 않습니다.

**강조해야 하는 내용**

- 오늘은 Astro 라우팅 전체 문법을 배우지 않습니다.
- `src/pages`와 URL의 연결만 이해하면 됩니다.
- 파일 위치와 이름이 중요합니다.

</details>

<details><summary>발표자 노트</summary>

Astro의 전체 라우팅 문법으로 깊게 들어가지 않습니다.
오늘은 `src/pages` 안의 파일이 주소가 된다는 감각만 잡으면 됩니다.

`about.astro` 파일이 생기면 `/about` 주소가 열립니다.
페이지가 404가 나면 파일 이름이나 위치를 먼저 확인하면 된다고 연결합니다.

</details>
