# Slide 26. 마지막에는 기록합니다

실습 마지막에는 구현을 더 늘리지 않고 기록합니다.

남길 것:

- 열린 URL과 안 열린 URL
- build 성공 결과 또는 실패 로그
- Projects / Posts가 어디까지 됐는지
- AGENTS.md 갱신 여부 또는 갱신 프롬프트
- 3회차에서 고치고 싶은 화면
- 참고하고 싶은 홈페이지 레퍼런스

이 기록이 있으면 다음 세션에서 바로 이어갈 수 있습니다.

<details><summary>슬라이드 모습과 강조점</summary>

**슬라이드 모습**

- 마지막에 남길 기록을 체크리스트처럼 보여줍니다.
- `구현을 더 늘리지 않고 기록` 문장을 강조합니다.
- 3회차 레퍼런스 준비가 빠지지 않게 화면에 포함합니다.

**강조해야 하는 내용**

- 마지막 10분에는 새 기능보다 기록이 우선입니다.
- AGENTS.md는 파일 수정까지 못 해도 갱신 프롬프트를 남기면 됩니다.
- 3회차를 위해 고치고 싶은 화면과 레퍼런스를 남깁니다.

</details>

<details><summary>발표자 노트</summary>

워크북 11단계와 연결되는 슬라이드입니다.
마지막에는 구현 욕심을 줄이고 기록으로 넘어갑니다.

AGENTS.md는 실제 파일까지 고치면 좋지만 필수 완주 조건은 아닙니다.
시간이 부족하면 갱신할 프롬프트만 남겨도 됩니다.

3회차에서는 화면의 특정 요소나 영역을 기준으로 디자인 수정을 요청할 예정입니다.
그래서 현재 화면에서 고치고 싶은 부분과 참고하고 싶은 홈페이지 레퍼런스를 남기게 합니다.

</details>
