# Slide 24. 검증 후 GitHub에 올립니다

build가 성공하면 배포 흐름을 확인합니다.

```text
commit
→ push
→ GitHub Actions 확인
→ GitHub Pages 확인
```

각 단어는 이렇게만 이해해도 됩니다.

```text
commit = 변경사항 저장점
push = GitHub에 올리기
Actions = 배포 작업이 도는 곳
Pages = 방문자가 보는 실제 주소
```

실패하면 화면을 고치기보다 로그를 먼저 남깁니다.

<details><summary>발표자 노트</summary>

워크북 10단계와 연결되는 슬라이드입니다.
로컬에서 보인다고 끝이 아니라, GitHub에 올라가고 Pages에서 확인되는지까지 봅니다.

commit은 변경사항의 저장점이고, push는 그 저장점을 GitHub에 올리는 일입니다.
GitHub Actions는 배포 작업이 돌아가는 곳이고, GitHub Pages는 방문자가 보는 실제 주소입니다.

다만 시간이 부족하면 여기까지 못 가도 실패가 아닙니다.
build 결과, git status, Actions 실패 화면, Pages 반영 상태 중 하나라도 기록하면 다음에 이어갈 수 있습니다.
