# Slide 11A. 데모를 볼 때 확인할 것

데모에서 코드를 모두 이해할 필요는 없습니다.

아래 네 가지만 봅니다.

1. AI가 바로 수정하지 않고 먼저 계획을 말하는가
2. 어떤 파일을 만들거나 수정하는지 알려주는가
3. 브라우저에서 실제 페이지를 확인하는가
4. build 후 commit / push가 필요한지 확인하는가

---

## 데모 중 멈춰야 하는 신호

- 새 라이브러리를 마음대로 설치하려고 함
- GitHub Pages 설정을 바꾸려고 함
- 기존 Home 내용을 전부 지우려고 함
- 외부 콘텐츠 API나 token을 지금 붙이려고 함

이럴 때는 먼저 멈추고 범위를 다시 말합니다.

<details><summary>발표자 노트</summary>

**슬라이드 화면 구성**

- 확인할 것 4개: 계획, 수정 파일, Browser 확인, build 후 commit/push 판단
- 멈춰야 하는 신호 4개: 새 라이브러리, Pages 설정, Home 삭제, 외부 API/token
- 화면은 체크리스트처럼 보이게 두면 됩니다.

**말할 내용**

데모를 볼 때 코드를 전부 읽으려고 하면 금방 지칩니다.
여기서는 코드를 이해하는 것보다, AI와 일할 때 무엇을 확인해야 하는지 보는 시간이라고 안내합니다.

확인할 것은 단순합니다.
계획을 먼저 말했는지, 어떤 파일을 바꾸는지, 브라우저와 build로 확인하는지입니다.

멈춰야 하는 신호는 AI를 못 믿으라는 뜻이 아닙니다.
범위를 다시 말해서 기존 배포 설정, secret, Home 내용 같은 중요한 것을 지키기 위한 안전장치라고 설명합니다.
