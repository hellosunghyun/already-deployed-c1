# Slide 16. SwiftUI와 비교하면 이동의 문제입니다

웹과 SwiftUI는 완전히 같지 않습니다.

하지만 처음에는 이렇게 비교할 수 있습니다.

```text
웹: /about 주소를 열면 About 페이지가 보임
SwiftUI: NavigationLink를 누르면 AboutView가 보임
```

공통점:

```text
사용자가 어디로 이동할 수 있는지 정해둔다
```

차이점:

```text
웹은 주소가 중요하고
앱은 화면 흐름이 중요합니다
```

<details><summary>슬라이드 모습과 강조점</summary>

**슬라이드 모습**

- 웹과 SwiftUI 비교를 두 줄로만 보여줍니다.
- 공통점과 차이점을 각각 코드블록으로 분리합니다.
- SwiftUI 고급 개념은 화면에 올리지 않습니다.

**강조해야 하는 내용**

- SwiftUI 비교는 NavigationLink와 View 정도로만 둡니다.
- 웹은 URL이 명확한 기준입니다.
- 오늘은 라우팅을 "이동할 수 있는 길"로 이해하면 됩니다.

</details>

<details><summary>발표자 노트</summary>

Swift 앱을 조금 해본 참가자에게는 NavigationLink와 AboutView 정도로 비교합니다.
하지만 완전히 같은 개념이라고 말하지는 않습니다.

웹사이트는 브라우저 주소창의 URL이 중요합니다.
iPhone 앱은 앱 안에서 화면이 이동하는 흐름이 중요합니다.

오늘은 둘 다 사용자가 어디로 이동할 수 있는지 정해두는 구조라는 정도만 연결합니다.

</details>
