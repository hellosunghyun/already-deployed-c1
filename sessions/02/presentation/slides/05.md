# Slide 05. 1회차 결과를 버리지 않습니다

1회차의 흐름은 아래였습니다.

```text
만든다
→ 확인한다
→ 저장한다
→ 배포한다
→ 링크로 본다
```

2회차에서는 같은 프로젝트를 유지합니다.

```text
새로 만들기
```

보다

```text
현재 프로젝트에 구조 추가하기
```

가 중요합니다.

<details><summary>슬라이드 모습과 강조점</summary>

**슬라이드 모습**

- 1회차 흐름을 위에 두고, 2회차의 요청 방향을 아래에 둡니다.
- `새로 만들기`와 `현재 프로젝트에 구조 추가하기`를 대비합니다.
- "현재 프로젝트 유지"가 한눈에 보이게 합니다.

**강조해야 하는 내용**

- 오늘은 새 프로젝트를 만드는 시간이 아닙니다.
- 배포 설정과 기존 Home 내용을 지키면서 구조를 추가합니다.
- AI에게도 이 맥락을 먼저 말해야 합니다.

</details>

<details><summary>발표자 노트</summary>

AI에게 "홈페이지 새로 만들어줘"라고 하면 기존 설정이나 1회차 결과가 사라질 수 있습니다.
오늘은 새로 만드는 요청보다 이어서 작업하는 요청이 중요합니다.

참가자가 자기 프로젝트에서 시작하든 예제 폴더에서 시작하든, 기준은 같습니다.
현재 프로젝트를 먼저 이해하고, 필요한 구조만 추가합니다.

이 슬라이드에서 "버리지 않고 이어간다"는 기준을 잡아둡니다.

</details>
