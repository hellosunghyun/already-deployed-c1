# Slide 09A. Component를 코드 없이 이해하기

Component는 **반복해서 쓰는 화면 틀**입니다.

프로젝트 카드가 3개라면 아래 두 가지를 나눠 생각합니다.

```text
틀: 카드 모양
내용: 프로젝트 이름, 설명, 역할, 도구, 결과, slug
```

카드 모양은 한 번만 정하고, 내용만 바꿔 끼웁니다.

---

## 왜 초보자에게 중요할까요?

나쁜 방식:

```text
카드 1 HTML 복사
카드 2 HTML 복사
카드 3 HTML 복사
```

좋은 방식:

```text
카드 틀 하나
src/data/projects.json에 프로젝트 정보 여러 개
slug로 상세 페이지 주소 만들기
반복해서 보여주기
```

예를 들어 이렇게 말합니다.

```text
카드 모양은 한 번 정해두고,
프로젝트가 늘어나면 src/data/projects.json에 정보 하나를 더 추가하는 방식입니다.
```

코드를 다 이해하지 못해도, 어디를 바꾸면 프로젝트가 추가되는지만 알면 충분합니다.
