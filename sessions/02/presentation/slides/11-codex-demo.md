# Slide 11. Codex 데모 흐름

오늘 데모는 전체 구현이 아니라 안전한 요청 방식만 보여줍니다.

## 보여줄 것

1. 현재 프로젝트 상태 확인 요청
2. `/about`, `/projects`, `/posts` 계획 받기
3. Layout과 Navigation 계획 받기
4. 파일 수정 승인하기
5. `npm run dev`로 확인하기
6. Browser로 페이지 이동 확인하기
7. `npm run build`로 검증하기
8. commit 메시지 추천 받기

---

## 오늘도 먼저 할 말

```text
아직 파일을 수정하지 말고, 먼저 계획을 설명해줘.
내가 승인하면 진행해줘.
```

<details><summary>말할 포인트</summary>

데모에서 가장 강조할 것은 코드를 빠르게 쓰는 장면이 아닙니다.
AI가 바로 파일을 고치기 전에 계획을 말하게 하는 습관입니다.

2회차는 수정 파일이 많아지기 때문에, 계획을 확인하지 않으면 어느 파일이 바뀌는지 놓치기 쉽습니다.
데모에서 전체 구현을 끝내려고 하지 않습니다.
참가자가 따라 볼 것은 "계획 요청 → 승인 → Browser 확인 → build 확인 → 기록"의 흐름입니다.

</details>
