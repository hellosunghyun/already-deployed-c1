# Slide 19. Component는 반복되는 화면 틀입니다

Projects 페이지에는 비슷한 카드가 여러 개 나옵니다.

카드마다 들어갈 내용:

```text
프로젝트 이름
한 줄 설명
내 역할
사용 도구
결과 또는 배운 점
자세히 보기
```

카드 모양은 한 번 만들고, 내용만 바꿔 끼우면 됩니다.

```text
ProjectCard = 카드 모양
projects.json = 카드에 들어갈 내용 목록
```

<details><summary>슬라이드 모습과 강조점</summary>

**슬라이드 모습**

- 카드에 들어갈 항목을 먼저 보여줍니다.
- 아래에 `ProjectCard`와 `projects.json` 역할을 분리합니다.
- 문법보다 "틀과 내용 분리"가 보이게 합니다.

**강조해야 하는 내용**

- Component는 반복해서 쓰는 화면 틀입니다.
- 카드 HTML을 계속 복사하지 않기 위해 씁니다.
- SwiftUI에서는 `ProjectCardView` 정도로 비교할 수 있습니다.

</details>

<details><summary>발표자 노트</summary>

Component라는 단어가 어렵게 들릴 수 있습니다.
오늘은 반복해서 쓰는 화면 틀이라고 설명하면 충분합니다.

프로젝트 카드가 3개 있다고 해서 카드 HTML을 3번 복사하는 것이 아닙니다.
카드 모양은 하나로 만들고, 프로젝트 내용만 바꿔 끼웁니다.

SwiftUI를 아는 참가자에게는 `ProjectCardView` 같은 작은 View를 떠올리면 된다고 말합니다.

</details>
