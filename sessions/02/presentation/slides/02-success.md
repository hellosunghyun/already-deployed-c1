# Slide 02. 오늘의 성공 기준

오늘은 많은 것을 다룹니다.

- Home / About / Projects / Posts
- Layout과 Navigation
- Projects 카드
- Posts 샘플 목록
- build / commit / push / deploy 확인
- 3회차 디자인 수정 재료 기록

하지만 성공 기준은 한 가지가 아닙니다.

## 최상

- Home / About / Projects / Posts 페이지가 있음
- 모든 페이지에 같은 Navigation이 있음
- Projects 카드가 3개 이상 있음
- Projects 카드에서 상세정보 또는 상세 링크로 이동 가능
- Posts 샘플 글 목록이 있음
- `npm run build` 성공
- GitHub Pages에 반영됨
- 3회차에서 고칠 디자인 포인트와 레퍼런스가 기록됨

## 표준

- 로컬에서 `/about`, `/projects`, `/posts`가 열림
- Layout과 Navigation이 적용됨
- Projects / Posts의 기본 데이터 또는 초안이 있음
- build 성공 또는 실패 로그가 있음
- 다음 세션에서 이어갈 질문이 기록됨

## 최소

- 현재 프로젝트 상태를 확인함
- 어디서 막혔는지 기록함
- 열린 URL과 안 열린 URL을 구분함
- build 실패 로그 또는 현재 화면 캡처가 있음
- 다음 질문을 만들 수 있음

---

오늘의 핵심은 완성도가 아니라
**구조, 검증, 기록**입니다.
