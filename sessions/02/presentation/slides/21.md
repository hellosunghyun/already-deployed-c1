# Slide 21. slug는 주소에 쓰는 짧은 이름입니다

`slug`는 제목이 아니라 주소용 이름입니다.

예:

```text
프로젝트 제목: 첫 개인 홈페이지
slug: first-homepage
상세 주소: /projects/first-homepage
```

왜 쓰나요?

- 한글 제목보다 주소로 쓰기 쉽습니다.
- 카드에서 상세 페이지 주소를 만들 수 있습니다.
- 나중에 항목이 늘어나도 연결 기준이 됩니다.

<details><summary>슬라이드 모습과 강조점</summary>

**슬라이드 모습**

- 제목, slug, 상세 주소 예시를 크게 보여줍니다.
- 이유는 세 개만 짧게 둡니다.
- `제목이 아니라 주소용 이름` 문장을 강조합니다.

**강조해야 하는 내용**

- slug는 초보자에게 낯선 단어이므로 꼭 짚고 넘어갑니다.
- 오늘은 복잡한 동적 라우팅을 몰라도 됩니다.
- 카드에서 상세 페이지로 이동할 때 주소를 만드는 기준입니다.

</details>

<details><summary>발표자 노트</summary>

slug는 그냥 넘어가면 헷갈리기 쉽습니다.
제목이 아니라 주소에 쓰는 짧은 이름이라고 말합니다.

예를 들어 제목이 "첫 개인 홈페이지"여도 주소에는 `first-homepage` 같은 짧은 영어 이름을 씁니다.
오늘은 이것이 상세 페이지로 이동하는 기준 정도로만 이해하면 됩니다.

동적 라우팅 심화는 오늘 다루지 않습니다.

</details>
