# Slide 13. 페이지와 섹션은 다릅니다

오늘 가장 헷갈릴 수 있는 단어입니다.

```text
페이지 = 주소가 따로 있는 화면
섹션 = 한 페이지 안의 구역
```

예를 들어:

```text
/about = 페이지
Home 안의 관심 분야 = 섹션
```

AI에게 요청할 때도 결과가 달라집니다.

<details><summary>슬라이드 모습과 강조점</summary>

**슬라이드 모습**

- 페이지와 섹션 정의를 가장 크게 둡니다.
- `/about`과 `Home 안의 관심 분야` 예시를 바로 아래에 둡니다.
- 마지막에는 AI 요청 결과가 달라진다고 연결합니다.

**강조해야 하는 내용**

- 주소가 따로 있으면 페이지입니다.
- 한 페이지 안의 덩어리면 섹션입니다.
- 오늘 만드는 것은 주로 주소가 따로 있는 페이지입니다.

</details>

<details><summary>발표자 노트</summary>

페이지와 섹션을 처음부터 정확히 구분하기는 어렵습니다.
여기서는 주소가 따로 있느냐로 판단하게 합니다.

`/about`처럼 브라우저 주소가 따로 있으면 페이지입니다.
Home 안에 들어가는 관심 분야나 연락처 블록은 섹션입니다.

이 구분이 잡히면 AI에게 요청할 때 결과가 훨씬 덜 흔들립니다.

</details>
