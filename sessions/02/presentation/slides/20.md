# Slide 20. JSON은 정보 파일입니다

JSON은 여러 정보를 정해진 모양으로 모아둔 파일입니다.

오늘은 표 같은 정보 파일로 이해해도 됩니다.

```text
src/data/projects.json = 프로젝트 정보 목록
src/data/posts.json    = 글 정보 목록
```

JSON을 쓰는 이유:

- 화면 코드에 문장을 흩뿌리지 않기
- 항목을 나중에 추가하기 쉽게 하기
- 외부 데이터가 들어와도 화면용 모양으로 바꿔 쓰기

<details><summary>슬라이드 모습과 강조점</summary>

**슬라이드 모습**

- JSON을 "정보 파일"로 정의합니다.
- `projects.json`과 `posts.json`의 역할을 나란히 보여줍니다.
- 이유는 세 개만 둡니다.

**강조해야 하는 내용**

- JSON 문법을 외우는 시간이 아닙니다.
- 여러 정보를 한 파일에 모아둔다는 감각이 중요합니다.
- 나중에 외부 데이터를 붙일 때도 화면용 JSON 모양으로 바꿔 쓰면 됩니다.

</details>

<details><summary>발표자 노트</summary>

JSON 문법을 자세히 설명하지 않습니다.
처음에는 표 같은 정보 파일이라고 생각해도 됩니다.

중요한 것은 카드나 글 목록에 들어갈 문장을 화면 코드 여기저기에 흩뿌리지 않는 것입니다.
정보 파일을 따로 두면 나중에 프로젝트나 글을 추가할 때 어디를 고치면 되는지 분명해집니다.

</details>
