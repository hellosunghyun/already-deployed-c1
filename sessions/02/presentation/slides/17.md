# Slide 17. Layout은 공통 바깥 틀입니다

여러 페이지에는 같은 Header, Navigation, Footer가 필요합니다.

나쁜 방식:

```text
about.astro에 Header 복붙
projects.astro에 Header 복붙
posts.astro에 Header 복붙
```

좋은 방식:

```text
Layout 하나를 만들고
모든 페이지가 그 Layout을 사용
```

Layout은 여러 페이지가 같이 입는 공통 옷입니다.

<details><summary>슬라이드 모습과 강조점</summary>

**슬라이드 모습**

- 나쁜 방식과 좋은 방식을 위아래로 비교합니다.
- 마지막에 `공통 옷` 비유를 짧게 둡니다.
- `<slot />` 같은 문법은 이 슬라이드에서 깊게 다루지 않습니다.

**강조해야 하는 내용**

- Layout은 여러 페이지가 같이 쓰는 바깥 틀입니다.
- Header와 Footer를 여러 파일에 복사하지 않기 위해 씁니다.
- SwiftUI 비교는 공통 View 재사용 정도로만 설명합니다.

</details>

<details><summary>발표자 노트</summary>

Layout은 여러 페이지가 같이 쓰는 바깥 틀이라고 말하면 충분합니다.
오늘은 디자인 시스템을 만드는 날이 아닙니다.

Header와 Navigation을 여러 페이지에 복사하면 나중에 고치기 어렵습니다.
한 곳에 모아두면 모든 페이지가 같은 사이트처럼 보이고, 수정도 쉬워집니다.

SwiftUI에는 Layout 프로토콜이라는 고급 개념도 있지만 오늘은 그 이야기가 아닙니다.
공통 View를 만들어 여러 화면에서 다시 쓰는 감각 정도로만 연결합니다.

</details>
