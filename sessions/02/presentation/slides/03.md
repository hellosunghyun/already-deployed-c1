# Slide 03. 오늘 다루는 범위

오늘은 많은 것을 다룹니다.

- Home / About / Projects / Posts
- 페이지와 섹션
- Astro 라우팅
- Layout과 Navigation
- Component와 JSON 데이터
- Projects 카드와 상세 링크
- Posts 샘플 목록
- Browser 확인과 build
- commit / push / deploy 확인
- 결과 기록과 다음 질문

범위는 넓지만, 모두가 같은 완성도로 끝내야 한다는 뜻은 아닙니다.

<details><summary>슬라이드 모습과 강조점</summary>

**슬라이드 모습**

- 오늘 만날 단어를 목록으로 보여줍니다.
- 마지막 문장은 목록과 분리해서 크게 둡니다.
- 너무 많은 내용을 한 번에 설명하지 말고 "앞으로 만날 지도"처럼 보여줍니다.

**강조해야 하는 내용**

- 발표자료는 오늘 만날 단어를 미리 보는 시간입니다.
- 범위가 넓다는 사실을 숨기지 않습니다.
- 넓은 범위와 성공 기준은 다르다고 바로 연결합니다.

</details>

<details><summary>발표자 노트</summary>

오늘 나오는 단어가 많다는 것을 먼저 인정합니다.
페이지, 라우팅, Layout, Component, JSON, build 같은 단어가 한 번에 나오면 어렵게 느껴질 수 있습니다.

하지만 발표 시간에 이 단어를 전부 구현하는 것은 아닙니다.
이후 워크북에서 다시 만날 단어를 미리 익히는 시간입니다.
