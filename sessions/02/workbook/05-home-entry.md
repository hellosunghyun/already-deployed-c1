# 05. Home 페이지를 입구로 정리하기

## 목표

Home 페이지를 전체 사이트의 입구처럼 정리합니다.

## 왜 하는가

Home은 모든 내용을 길게 넣는 페이지가 아닙니다.
처음 온 사람이 "이 사이트에서 무엇을 볼 수 있는지" 빠르게 이해하고 About / Projects / Posts로 이동할 수 있어야 합니다.

## 쉬운 설명

Home에 필요한 것은 많지 않습니다.

아래 기준은 이번 수업에서 쓰는 출발점입니다.
모든 개인 홈페이지가 반드시 이렇게 시작해야 한다는 뜻은 아닙니다.

- 이름 또는 닉네임
- 한 줄 소개
- 관심 분야 2~3개
- About / Projects / Posts로 가는 안내
- 너무 부담스럽지 않은 첫 문장

## 내 정보 정리

아직 문장이 준비되지 않았다면 아래를 임시로 채웁니다.

```text
이름 또는 닉네임:
한 줄 소개:
관심 분야 1:
관심 분야 2:
관심 분야 3:
방문자가 먼저 봤으면 하는 페이지: About / Projects / Posts
```

템플릿은 [개인 정보 정리 템플릿](../templates/personal-intake.md)을 참고하세요.

## 공통 프롬프트

```text
Home 페이지를 개인 홈페이지의 입구로 정리하고 싶어.

현재 상태:
- Astro 프로젝트가 있다.
- Home / About / Projects / Posts 페이지가 있다.
- 공통 Layout과 Navigation이 있다.

내 정보:
- 이름 또는 닉네임: [여기에 입력]
- 한 줄 소개: [여기에 입력]
- 관심 분야: [예: AI, 디자인, iOS, 웹, 데이터, 교육 등]
- 방문자에게 먼저 보여주고 싶은 것: [About / Projects / Posts 중 선택]

목표:
- Home을 전체 사이트의 첫인상으로 정리한다.
- 방문자가 내가 누구인지 대략 알 수 있게 한다.
- About / Projects / Posts로 이동할 수 있는 안내를 넣는다.
- 모든 내용을 Home에 다 넣지 않는다.
- 이 구성은 이번 수업의 기준이며, 나중에 바꿀 수 있다는 점을 설명한다.

문장 톤:
- 과장하지 않기
- 처음 보는 사람도 이해할 수 있게 쓰기
- 짧은 문장과 단락 사용
- "열정적인", "혁신적인" 같은 추상적인 표현을 남발하지 않기
- 아직 확정되지 않은 내용은 임시 문장으로 자연스럽게 처리하기

디자인:
- 기존 Layout과 어울리게
- 흰 배경과 넓은 여백 유지
- 큰 제목은 한 번만 사용
- 모바일에서도 읽기 쉽게
- 새 라이브러리 추가하지 않기

작업 방식:
1. 먼저 현재 Home 파일을 확인해줘.
2. 어떤 섹션으로 나누면 좋을지 제안해줘.
3. 수정할 파일 목록을 말해줘.
4. 아직 수정하지 말고 계획만 설명해줘.
5. 내가 승인하면 진행해줘.

완료 후 알려줄 것:
1. 변경한 파일 목록
2. Home에 들어간 섹션 목록
3. 내가 나중에 바꾸면 좋은 문장
4. 브라우저에서 확인할 주소
```

## 승인 후 추가 프롬프트

```text
좋아. Home 페이지를 방금 제안한 구조로 정리해줘.

주의:
- About / Projects / Posts 페이지 내용은 아직 크게 바꾸지 마.
- Home이 너무 긴 자기소개서처럼 보이지 않게 해줘.
- 링크 텍스트는 초보자도 이해할 수 있게 명확하게 써줘.
```

## 체크박스

- [ ] Home에 이름 또는 닉네임이 보인다.
- [ ] 한 줄 소개가 보인다.
- [ ] 관심 분야가 2~3개 보인다.
- [ ] About으로 가는 안내가 있다.
- [ ] Projects로 가는 안내가 있다.
- [ ] Posts로 가는 안내가 있다.
- [ ] Home이 너무 길지 않다.

## Ready Gate

다음 단계로 넘어가기 전에 아래가 되어 있어야 합니다.

- Home을 처음 본 사람이 다른 페이지로 이동할 수 있습니다.
- Home에 모든 내용을 다 넣으려 하지 않았습니다.
- 임시 문장이더라도 현재 자신을 설명하는 문장이 들어 있습니다.
