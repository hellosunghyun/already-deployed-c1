# 02. 현재 프로젝트 상태 점검

## 목표

파일을 수정하기 전에 현재 Astro 프로젝트 상태를 확인합니다.

## 왜 하는가

2회차는 여러 파일을 추가하고 수정합니다.
현재 상태를 모르고 시작하면 AI가 잘못된 폴더에서 작업하거나, 배포 설정을 망가뜨리거나, 기존 파일을 덮어쓸 수 있습니다.

## 쉬운 설명

오늘 확인할 것은 "이 프로젝트가 지금 어디까지 되어 있는가"입니다.

특히 아래를 확인합니다.

- `package.json`이 있는지
- `src/pages`가 있는지
- 현재 Git branch가 무엇인지
- 수정되지 않은 변경사항이 이미 있는지
- 마지막 commit이 무엇인지
- 로컬 실행이 가능한지

## 공통 프롬프트

이 프롬프트는 일부러 자세합니다.
길다고 줄이기보다, 현재 상태와 확인 기준을 빠뜨리지 않기 위해 그대로 복사해서 사용하세요.

```text
2회차 구조 작업을 시작하기 전에 현재 Astro 프로젝트 상태를 점검해줘.

목표:
- 잘못된 폴더에서 작업하지 않기
- 기존 1회차 작업을 망가뜨리지 않기
- 수정 전 기준 상태를 확인하기

확인할 것:
1. 현재 작업 폴더 경로
2. package.json 존재 여부
3. src/pages 폴더 구조
4. astro.config.mjs 존재 여부와 site 설정
5. git 저장소 여부
6. 현재 branch
7. git status 결과
8. 마지막 commit
9. npm run dev 또는 npm run build를 실행할 수 있는 상태인지

중요:
- 아직 파일을 수정하지 마.
- 삭제나 reset 같은 위험한 명령은 실행하지 마.
- 먼저 어떤 명령어로 확인할지 계획을 설명해줘.
- 내가 승인하면 확인 명령어를 실행해줘.
- 확인 후 "2회차 작업을 시작해도 되는지"를 판단해줘.
```

## 확인 후 AI에게 추가로 요청할 말

```text
방금 확인한 결과를 초보자도 이해할 수 있게 요약해줘.

아래 형식으로 알려줘.

1. 현재 프로젝트가 맞는지
2. 바로 작업해도 되는지
3. 먼저 commit해야 할 변경사항이 있는지
4. 오늘 조심해야 할 파일
5. 다음 단계에서 만들 파일 예상 목록
```

## 체크박스

- [ ] 현재 폴더 경로를 확인했다.
- [ ] `package.json`이 있는지 확인했다.
- [ ] `src/pages` 구조를 확인했다.
- [ ] `git status`를 확인했다.
- [ ] 마지막 commit을 확인했다.
- [ ] 작업 전 commit이 필요한지 판단했다.

## Ready Gate

다음 단계로 넘어가기 전에 아래가 되어 있어야 합니다.

- 현재 프로젝트 폴더가 맞습니다.
- 기존 변경사항이 있으면 commit했거나, 왜 남겨두는지 알고 있습니다.
- AI가 다음에 만들 파일 목록을 예상했습니다.

## 힌트

<details>
<summary>Hint 1</summary>

`package.json`이 보이지 않는다면 프로젝트 루트가 아닐 수 있습니다.

</details>

<details>
<summary>Hint 2</summary>

`git status`에 변경 파일이 이미 많다면 바로 작업하지 말고 먼저 AI에게 "이 변경사항이 무엇인지" 물어보세요.

</details>

<details>
<summary>Hint 3</summary>

`astro.config.mjs`의 `site` 값은 GitHub Pages 배포와 관련될 수 있습니다. 오늘 구조 작업 중 실수로 지우지 않게 합니다.

</details>
