# 01. 1회차 결과 확인

## 목표

2회차를 시작할 수 있는 기준 상태를 확인합니다.

## 왜 하는가

참가자마다 1회차 결과가 다릅니다.
누군가는 `github.io` 링크까지 만들었고, 누군가는 로컬 실행까지만 되었고, 누군가는 예제 폴더에서 시작해야 할 수 있습니다.

출발점을 먼저 알아야 AI에게 정확한 맥락을 줄 수 있습니다.

## 쉬운 설명

오늘은 세 가지 출발 경로가 있습니다.

| 경로 | 상태 | 오늘 할 일 |
|---|---|---|
| A | GitHub Pages 링크가 열림 | 본인 프로젝트에서 바로 시작 |
| B | GitHub 업로드는 됐지만 Pages가 안 됨 | 구조 작업 후 배포 문제 재확인 |
| C | 1회차를 못 끝냄 | 1회차 완료 예제로 시작 |

## 확인할 것

아래 값을 적어둡니다.

```text
GitHub 저장소 URL:
GitHub Pages URL:
로컬 프로젝트 폴더 위치:
사용할 AI 도구: Codex / Claude Code / Antigravity
오늘 출발 경로: A / B / C
```

## 공통 프롬프트

이 프롬프트는 일부러 자세합니다.
길다고 줄이기보다, 현재 상태와 확인 기준을 빠뜨리지 않기 위해 그대로 복사해서 사용하세요.

```text
2회차 작업을 시작하려고 해.
1회차에서 만든 Astro 개인 홈페이지를 이어서 작업하는 것이 목표야.

먼저 내 출발 상태를 정리해줘.

내가 알고 있는 정보:
- GitHub 저장소 URL: [여기에 입력]
- GitHub Pages URL: [여기에 입력, 없으면 없음]
- 로컬 프로젝트 폴더 위치: [여기에 입력]
- 사용하는 AI 도구: [Codex / Claude Code / Antigravity]
- 1회차 완료 상태: [github.io 링크 열림 / GitHub 업로드만 됨 / 로컬 실행만 됨 / 아직 못 끝냄]

확인해줬으면 하는 것:
1. 지금 2회차를 바로 시작해도 되는지
2. 먼저 복구하거나 확인해야 할 것이 있는지
3. 오늘 어떤 경로로 진행하면 좋을지
4. 내가 절대 지우면 안 되는 파일이나 설정이 무엇인지

중요:
- 아직 파일을 수정하지 마.
- 먼저 현재 상태와 권장 진행 경로만 설명해줘.
- 내가 승인하기 전에는 명령어를 실행하지 마.
```

## 1회차 완료 예제로 시작해야 하는 경우

1회차를 끝내지 못했다면 [`../01/example-homepage`](../../01/example-homepage)를 기준 상태로 사용할 수 있습니다.

AI에게 이렇게 요청하세요.

```text
나는 1회차를 끝까지 완료하지 못해서,
2회차는 제공된 1회차 완료 예제에서 시작하려고 해.

예제 폴더:
sessions/01/example-homepage

내 목표:
- 이 예제 폴더를 2회차 실습 기준 상태로 이해하기
- 파일 구조를 먼저 설명받기
- 필요한 패키지를 설치하고 로컬에서 실행 가능한지 확인하기
- 이후 Home / About / Projects / Posts 구조를 추가하기

조건:
- 아직 파일을 수정하지 마.
- 먼저 예제 폴더 구조와 실행 계획을 설명해줘.
- 내가 승인하면 설치와 실행 확인을 진행해줘.
```

## 체크박스

- [ ] GitHub 저장소 URL을 알고 있다.
- [ ] GitHub Pages URL이 열리는지 확인했다. 또는 아직 없다는 것을 확인했다.
- [ ] 로컬 프로젝트 폴더 위치를 알고 있다.
- [ ] 오늘 출발 경로 A / B / C 중 하나를 정했다.
- [ ] AI가 현재 상태를 먼저 요약했다.

## Ready Gate

다음 단계로 넘어가기 전에 아래가 되어 있어야 합니다.

- 작업할 프로젝트 폴더가 정해졌습니다.
- 오늘 진행 경로가 정해졌습니다.
- AI가 아직 파일을 수정하지 않았습니다.

## 힌트

<details>
<summary>Hint 1</summary>

GitHub Pages URL이 안 열려도 2회차를 시작할 수 있습니다. 로컬 프로젝트가 있고 `npm run dev`가 되면 구조 작업은 가능합니다.

</details>

<details>
<summary>Hint 2</summary>

어떤 폴더를 열어야 할지 모르겠다면 `package.json`이 있는 폴더를 찾으세요.

</details>

<details>
<summary>Hint 3</summary>

예제 폴더에서 시작하는 것은 실패가 아닙니다. 워크숍에서는 같은 기준 상태에서 다시 출발하는 것이 더 안전할 때가 있습니다.

</details>
